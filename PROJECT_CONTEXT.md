## 这个工程是什么

这是一个专门为四年级数学课外拓展教学设计的**交互动画系统**。

服务对象：一名四年级孩子（Zoey），家长在课后辅导，配合 Claude Project 形成完整的教学闭环。

**这个工程在整体系统里的位置：**

```
Claude Project（教学大脑）
    ↓ 生成动画需求规格
Claude Code 工程（本工程）
    ↓ 开发并部署动画
GitHub Pages（在线访问）
    ↓ 链接嵌入
Notion（课件笔记库）
    ↓ 家长和孩子上课时使用
```

动画不是装饰，是教学的核心工具。每一个动画都服务于一个明确的教学目标——让孩子通过**操作**和**观察**自己推导出数学结论，而不是被动接受知识。

## 教学体系结构

### 核心教学原则

1. **从具体到抽象** — 先让孩子感知，再用自己的话描述，最后才是符号和公式
2. **孩子操作，不是家长演示** — 动画的控制权必须在孩子手里
3. **一个动画只呈现一个核心概念** — 不要在一个动画里塞多个知识点
4. **动画结束留白** — 演示完停在最终状态，给孩子思考时间

### 当前教材：四年级下册（人教版）

```
第1单元  四则运算          — 运算顺序的逻辑
第2单元  观察物体（二）    — 空间想象
第3单元  运算律            — 简便计算的底层原理
第4单元  小数的意义和性质  ← 当前单元
第5单元  三角形            — 几何推理
第6单元  小数的加法和减法  — 整数运算规律的迁移
第7单元  图形的运动（二）  — 空间变换
第8单元  平均数与条形统计图 — 数据分析
第9单元  数学广角——鸡兔同笼 — 逻辑推理
```

### 第4单元知识点（当前重点）

```
4-1 小数的意义        ← 当前知识点，整个单元的地基
4-2 小数的读法和写法
4-3 小数的性质
4-4 小数的大小比较
4-5 小数点移动引起小数大小的变化
4-6 解决问题
4-7 小数与单位换算
4-8 小数的近似数
4-9 整理和复习
```

**核心本质：** 小数是分数的另一种写法。0.1 = 1/10，0.01 = 1/100。所有动画都应该强化这个底层理解。

## 公共组件库规范（shared/utils.js）

所有动画必须优先使用公共组件，不要重复造轮子。新增动画时如果需要新组件，先添加到 utils.js，再在动画里调用。

### 已有组件

```javascript
NumberLine(config)
// 数轴组件，可缩放、可定位
// config: { start, end, step, zoom, highlight }

GridSquare(config)
// 网格正方形，可切割、可选中
// config: { rows, cols, color, selectedColor }

FractionDisplay(numerator, denominator)
// 分数显示，支持动画过渡

DecimalDisplay(value, highlightDigit)
// 小数显示，小数点位置可动

Arrow(from, to, label, config)
// 带标注的箭头

ScaleIndicator(multiplier)
// ×10 / ÷10 的放大缩小指示器
```

### 添加新组件的规范

- 组件名使用 PascalCase
- 必须支持动画过渡（不能瞬间变化）
- 必须支持移动端触摸操作
- 必须在 utils.js 顶部的组件索引注释里登记

## 动画开发规范

### 交互标准

- **孩子能控制的**：拖动、点击、滑块、按钮
- **不允许**：自动播放完整动画后孩子只能看，没有操作空间
- **必须有**：重置按钮，让孩子可以重新开始

### 视觉规范

- 配色使用 assets/style.css 里定义的 CSS 变量，不允许硬编码颜色
- 字体大小：标注文字 16px，数字显示 24px，核心概念 32px
- 动画速度：切割/移动类 600–800ms，数字变化类 400ms，不允许瞬间跳变
- 移动端适配：所有动画必须在平板上正常显示和操作

### 菜单结构

每新增一个知识点动画，必须同步更新 index.html 里的菜单：

```
第4单元 小数的意义和性质
  ├── 4-1 小数的意义
  ├── 4-2 小数的读法和写法
  └── ...
```

菜单项命名格式：`[序号] [知识点名称]`，和教材保持一致。

### 性能要求

- 首屏加载不超过 2 秒（GitHub Pages 网络环境）
- 动画帧率保持 60fps
- 单个知识点模块文件不超过 200KB

## 与 Claude Project 的协作方式

Claude Project 负责教学内容，Claude Code 负责技术实现。两者通过 `ANIMATION_SPEC.md` 交换信息。

**收到新任务时的标准流程：**

1. 先读 `PROJECT_CONTEXT.md`（本文件）确认全局上下文
2. 再读 `ANIMATION_SPEC.md` 里的具体需求
3. 检查 `shared/utils.js` 确认可复用的组件
4. 开发新的知识点模块
5. 更新 `index.html` 菜单
6. 部署到 GitHub Pages
7. 输出完成回执（链接 + 菜单路径），供家长告知 Claude Project

**不允许的行为：**
- 不读 PROJECT_CONTEXT.md 就开始开发
- 重新实现已有公共组件
- 使用和全局样式不一致的颜色/字体
- 只完成动画但不更新菜单
